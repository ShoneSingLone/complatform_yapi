### 1.3 provide/inject 状态管理机制

#### 1.3.1 实现方式

business_yapi 项目使用 Vue 的原生 `provide/inject` API 进行组件间状态传递，主要特点如下：

1. **层级传递**：通过组件树的层级关系，父组件使用 `provide()` 方法提供状态，子组件使用 `inject` 选项接收状态
2. **全局 APP 实例**：顶层组件通常会 provide `APP` 实例，子组件通过 inject 接收，实现全局状态访问
3. **组件实例传递**：父组件可以将自身实例 provide 给子组件，子组件可以直接调用父组件的方法和访问属性

#### 1.3.2 使用示例

**父组件提供状态：**
```vue
<script lang="ts">
export default async function () {
  return {
    provide() {
      return {
        inject_project: this,
        // 其他需要传递的状态或方法
      };
    },
    data() {
      return {
        // 组件状态
      };
    },
    methods: {
      // 组件方法
    }
  };
}
</script>
```

**子组件接收状态：**
```vue
<script lang="ts">
export default async function () {
  return {
    inject: ["APP", "inject_project"],
    methods: {
      someMethod() {
        // 使用注入的状态和方法
        console.log(this.inject_project.someData);
        this.inject_project.someMethod();
      }
    }
  };
}
</script>
```

#### 1.3.3 优势与限制

**优势：**
- 无需额外依赖，使用 Vue 原生 API
- 实现简单，代码结构清晰
- 适合中小型应用的状态管理需求
- 支持跨多层组件传递状态，避免 props 层层传递的问题

**限制：**
- 状态变更追踪困难，不利于调试
- 大型应用中可能导致状态管理混乱
- 缺乏中间件、模块化等高级特性

### 1.4 前端代码规则

**核心规则：**

1. **模块导出格式**：所有 .vue 文件使用 `export default async function ({ PRIVATE_GLOBAL })` 结构
   ```javascript
   export default async function ({ PRIVATE_GLOBAL }) {
     // 组件逻辑
     return {
       // 组件选项
     }
   }
   ```

2. **TypeScript 使用**：
   - 使用 `<script lang="ts">` 仅为 IDE 识别
   - 不经过构建处理，无法运行需要编译的 TypeScript 特性
   - 只能使用浏览器原生支持的 JavaScript 语法

3. **模块加载机制**：
   - 使用 `_.$importVue()` 动态加载组件
   - 支持 `@/` 前缀表示项目根目录
   - 支持 `/common/` 前缀表示公共依赖

4. **API 调用方式**：
   - 使用 `_api.yapi` 进行后端 API 调用
   - 遵循现有 API 调用模式
