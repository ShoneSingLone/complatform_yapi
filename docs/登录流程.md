
# 项目登录和鉴权流程总结

## 1. 前端登录流程

### 1.1 登录表单提交
- **文件**：`static/business_xspace/views/Login/LoginForm.vue`
- **流程**：
  1. 用户在登录表单中输入邮箱和密码
  2. 点击登录按钮或按回车键触发 `login()` 方法
  3. 表单验证通过后，调用 `_api.xspace.userLogin(formData)` 发送登录请求
  4. 登录成功后，将返回的 `x_token` 存储到本地存储：`_.$lStorage.x_token = res?.data?.x_token`
  5. 调用 `this.APP.refreshUserInfo()` 刷新用户信息
  6. 显示登录成功提示

### 1.2 API 请求拦截与 token 处理
- **文件**：`static/business_xspace/utils/api.vue`
- **流程**：
  1. 前端通过 `_.$ajax.requestInjector` 拦截所有 API 请求
  2. 调用 `Vue._common_utils.appendToken(req.url)` 为请求 URL 添加 token
  3. 通过 `_.$ajax.responseInjector` 处理响应
  4. 当响应码为 40011 或 10001 时，判断为登录过期，跳转到登录页

## 2. 后端登录处理

### 2.1 登录接口
- **文件**：`server/controllers/Autoware/api/User.ts`
- **接口**：`/api/user/login`
- **流程**：
  1. 接收邮箱和密码参数
  2. 通过 `userInst.findByEmail(email)` 查询用户信息
  3. 验证密码：`xU.$saltIt(password, userInfo.passsalt) === userInfo.password`
  4. 生成 JWT token：`jwt.sign({ uid }, passsalt)`
  5. 设置 cookie：
     - `_xspace_token`: JWT token
     - `_xspace_uid`: 用户 ID
  6. 返回用户信息和 token：
     ```json
     {
       "uid": "...",
       "username": "...",
       "role": "...",
       "email": "...",
       "x_token": {
         "_xspace_token": "...",
         "_xspace_uid": "..."
       }
     }
     ```

### 2.2 登出接口
- **文件**：`server/controllers/Autoware/api/User.ts`
- **接口**：`/api/user/logout`
- **流程**：
  1. 清除 cookie 中的 `_xspace_token` 和 `_xspace_uid`
  2. 返回成功响应

## 3. Token 处理机制

### 3.1 用户登录 Token
- **生成方式**：使用 JWT (`jsonwebtoken` 库)
- **存储方式**：
  - 后端：存储在 cookie 中
  - 前端：存储在本地存储 (`localStorage`)
- **验证方式**：后端通过 JWT 验证机制验证 token 有效性

### 3.2 项目 Token
- **文件**：`server/utils/token.ts`
- **生成方式**：使用 AES192 加密
  ```javascript
  exports.getToken = function getToken(token, uid) {
    if (!token) throw new Error("token 不能为空");
    xspace_configs.passsalt = xspace_configs.passsalt || defaultSalt;
    return aseEncode(uid + "|" + token, xspace_configs.passsalt);
  };
  ```
- **解析方式**：
  ```javascript
  exports.parseToken = function parseToken(token) {
    if (!token) throw new Error("token 不能为空");
    xspace_configs.passsalt = xspace_configs.passsalt || defaultSalt;
    let tokens;
    try {
      tokens = aseDecode(token, xspace_configs.passsalt);
    } catch (e) {}
    if (tokens && typeof tokens === "string" && tokens.indexOf("|") > 0) {
      tokens = tokens.split("|");
      return {
        uid: tokens[0],
        projectToken: tokens[1]
      };
    }
    return false;
  };
  ```
- **存储方式**：存储在 MongoDB 的 `token` 集合中
  - **文件**：`server/models/token.ts`
  - **结构**：
    ```javascript
    getSchema() {
      return {
        project_id: { type: Number, required: true },
        token: String
      };
    }
    ```

## 4. 鉴权流程

### 4.1 前端鉴权
- 所有 API 请求通过 `requestInjector` 自动添加 token
- 响应拦截器处理 token 过期情况，自动跳转到登录页

### 4.2 后端鉴权
- 后端接口通过 `auth: true` 标记需要鉴权的接口
- 后端中间件验证 token 有效性
- 验证通过后，将用户信息存储到 `this.$user` 中
- 接口处理函数可以通过 `this.getUid()` 和 `this.getRole()` 获取用户信息

## 6. 总结

1. **登录流程**：
   - 前端提交登录表单 → 后端验证 → 生成 JWT token → 存储到 cookie 和本地存储 → 前端获取用户信息

2. **鉴权流程**：
   - 前端请求自动携带 token → 后端验证 token → 授权访问或拒绝

3. **Token 类型**：
   - **用户登录 token**：JWT 格式，用于用户身份认证
   - **项目 token**：AES 加密，用于项目级别的认证

4. **安全措施**：
   - 密码加盐存储
   - JWT token 验证
   - Cookie 设置 `httpOnly: true`
   - 登录过期自动跳转

该项目采用了 JWT 作为主要的认证机制，结合 cookie 和本地存储来管理 token，确保了系统的安全性和用户体验的流畅性。
